Broccoli is a robust, high-performance message queue library for Rust applications. Inspired by the simplicity and power of Celery, Broccoli brings reliable asynchronous job processing to Rust with the benefits of compile-time type safety and zero-cost abstractions. Whether you're building microservices, distributed systems, or real-time data pipelines, Broccoli provides a simple yet flexible API for publishing, consuming, and processing messages across multiple message brokers.

## Why Broccoli?

If you're familiar with Celery but want to leverage Rust's performance and type safety, Broccoli is your answer. While Celery is battle-tested in Python ecosystems, Broccoli brings the same proven philosophy to Rust with:

- **Type Safety**: Messages are validated at compile-time, eliminating entire classes of runtime errors
- **Performance**: Leverages Rust's zero-cost abstractions and async/await for minimal overhead
- **Multi-Broker Flexibility**: Start with Redis for simplicity, switch to RabbitMQ for advanced features, or use SurrealDB for embedded scenarios—all with the same API
- **Simple API**: Intuitive builder patterns and familiar async/await semantics
