Broccoli is a high-performance message queue library for Rust applications, designed as a Rust-native alternative to Python's Celery. It provides a unified, type-safe API for publishing and consuming messages across multiple broker backends — currently Redis, RabbitMQ, and SurrealDB — with async-first design built on Tokio.

## Why Broccoli?

Broccoli brings Celery-style job queues to the Rust ecosystem. If you're building async Rust services and need reliable background job processing, Broccoli gives you:

  <Card title="Type-Safe Messages" icon="shield-check" href="/concepts/messages">

## Get Started

<Steps>
  <Step title="Add the dependency">
    Add `broccoli_queue` to your `Cargo.toml` with your chosen backend feature.

    ```toml Cargo.toml
    [dependencies]
    broccoli_queue = { version = "0.4", features = ["redis"] }
    tokio = { version = "1", features = ["full"] }
    serde = { version = "1", features = ["derive"] }
    ```
  </Step>
  <Step title="Build a queue">
    Connect to your broker using the fluent builder API.

    ```rust
    use broccoli_queue::queue::BroccoliQueue;

    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;
    ```
  </Step>
  <Step title="Publish a message">
    Define a serializable type and publish it to a named topic.

    ```rust
    use serde::{Deserialize, Serialize};

    #[derive(Clone, Serialize, Deserialize)]
    struct MyJob { id: String, task: String }

    queue.publish("jobs", None, &MyJob {
        id: "1".into(),
        task: "send_email".into(),
    }, None).await?;
    ```
  </Step>
  <Step title="Consume and process">
    Start a worker loop with configurable concurrency.

    ```rust
    use broccoli_queue::brokers::broker::BrokerMessage;

    queue.process_messages("jobs", Some(4), None,
        |msg: BrokerMessage<MyJob>| async move {
            println!("Processing: {}", msg.payload.task);
            Ok(())
        },
    ).await?;
    ```
  </Step>
</Steps>
